ACID:
atomic : 一个事务里面的操作要么全部成功，要么全部失败
consistent : 提交的事务对未来的事务可见
isolated : 未提交的事务相互之间不可见
durable : 数据被永久保存

对于ACI，可以有如下表述：
一个事务内一系列操作ops对数据datas进行操作
ops在提交之前，ops对datas产生的变化，对其他事务不可见
ops在提交之后，ops对datas产生的变化，对其他事务可见
ops要么全部成功，要么全部失败

一致性：
	客户端：多并发访问的情况下，更新过的数据如何获取
	服务器端：如何将更新复制到整个系统，保证数据最终一致
	一致性策略：强一致性、弱一致性、最终一致性
可用性：
	每一个非故障的节点必须对每一个请求作出响应
	也就是，系统中使用的任何算法都必须是最终终止的
	也就是说只要是节点能够被连上，那么该节点必须提供服务
分区容忍性：
	分布式系统中一个或者几个机器down了，或者机器之间的网络出现异常，将系统分割为独立的几个部分，各个部分还能维持分布式系统的运作。
